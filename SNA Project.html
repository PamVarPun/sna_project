สายสัมพันธ์ ณ ภพไหน ๆ

ด้วยชนชั้นอันต่างศักดิ์ในสมัยกรุงศรีอยุธยา

ในสมัยอยุธยามีความเกี่ยวข้องกับเรื่องของชนชั้นวรรณอย่างชัดเจน รายงานฉบับนี้จึงได้ใช้สีเพื่อการแบ่งกลุ่มชนชั้น และขนาดแต่ละ Node แสดง คำณวนตามจำนวน degree ที่แต่ละ Node มี ซึ่ง Node ที่มี degree มากที่สุดในความสัมพันธ์นี้คือ "เกศสุรางค์" โดยมี degree อยู่ที่ 22 degrees ด้วยกัน อีกทั้งแต่ละ edge มีค่า weigth ที่แตกต่างกันตามน้ำหนักความสัมพันธ์ โดยมีรายละเอียดดังนี้

ความสัมพันธ์ของแต่ละตัวละคร


ในตารางนี้จะมี Node สีพิเศษปรากฎขึ้น นั่นก็คือ "การฟื้นคืนชีพ" เนื่องด้วยละครบุพเพสันนิวาส และ พรหมลิขิต บทประพันธ์ของ รอมแพง ได้เขียนขึ้นตามความหมายของ "กงล้อธรรมจักร" หรือกงล้อแห่งธรรมตามความเชื่อแบบฉบับชาวพุทธและฮินดู ซึ่งหนึ่งในรายละเอียดประกอบด้วยการประดับแฉกของอาทิตย์ หรือ อทิตยา มีดอกพุดตานและดอกการะเกดคอยประดับ เป็นต้น โดยแต่ละ Node มีความสัมพันธ์เชื่อมโยงถึงกันในหลากหลายรูปแบบ ดังนี้

ละครเรื่องพรหมลิขิต เป็นภาคต่อของ บุพเพสันนิวาส โดยแต่ละภาคเป็นเหตุการณ์ที่เกิดขึ้นต่างยุคสมัย ซึ่งมีตัวละครบางกลุ่มที่อยู่ต่อเนื่องจากแผ่นดินแรกเรื่อยมากระทั่งแผ่นดินที่ 2 โดยมีการใช้สีเพื่อแบ่งกลุ่ม ดังนี้

การวิเคราะห์ความสัมพันธ์


Closeness Centrality

ค่าความสนิทชิดเชื้อ "Closeness" เป็นอัลกอริทึ่มที่ให้ความสำคัญกับการเชื่อมต่อ Node ในเครือข่ายได้อย่างรววดเร็วที่สุด โดยกราฟได้ใช้สีในการไล่ออ่อนจากเข้ม โดย Node ไหนที่มีสีอ่อนแสดงว่ามีค่าความ Closeness ที่ต่ำไล่ลำดับค่าสีขึ้นไปเรื่อย ๆ ทั้งนี้ค่า Close จะไม่เกี่ยวว่าต้องเป็น Node ที่มี degree สูง หากเป็น Node ที่มี degree ต่ำ แต่สามารถเชื่อมโยงกับ Node ที่สำคัญได้ก็จะได้ค่า Closeness ที่สูงนั่นเอง

Betweeness Centrality

"สะพานของความสัมพันธ์ที่สั้นที่สุด" ใช้หลักการคำณวนหา Node ที่มีระยะทางเชื่อมโยงไปยังอีกคนหลาย ๆ คนที่สั้นที่สุด เรียกว่าใครที่มีค่า Betweeness มากที่สุด คือคนกลางที่จะเชื่อมโยงกลุ่มความสัมพันธ์ได้สั้นมากที่สุด โดยกราฟนี้ใช้การแสดงไล่สีจากอ่อนไปเข้ม แสดงว่าใครที่ไม่ได้เชื่อมโยงถึงคนอื่นเท่าไหร่นัก จะมีสีที่จืดจาง ส่วนใครที่มีการเชื่อมต่อที่สำคัญระหนึ่งหากตัดคนนั้นแล้ว อาจไม่มีการเชื่อมต่อกันของกลุ่มความสัมพันนี้ จะมีสีเข้มมากที่สุด ในรายงานนี้หมายถึง เกศสุรางค์ นั่นเอง


HITS: Hyperlink-Induced Topic Search

Hub and Authority


Authority

แหล่งข้อมูลชั้นเลิศที่ใครก็อยากเข้าหา : Authority เป็นการวัดจำนวนการเชื่อมโยงเข้าหาโหนดนั้นๆ จากโหนดสำคัญในเครือข่าย โดยไม่จำเป้นต้องเป็น Node ที่มีจำนวน Degree มาก และต่างจากค่า Closeness ที่เน้นความรวดเร็วในการเชื่อมโยงกับ Node อื่นในเครือข่าย

Hub

เจ้าบ้านผู้น่ารัก : Hub วัดความสำคัญของ Node คล้ายกับ Authority แต่เป็นในรูปแบบตรงข้าม นั่นก็คือ ให้ความสำคัญกับ Node ในการเชื่อมโยงออกไปหา Node สำคัญอื่นๆ ซึ่งไม่ผันตรงกับจำนวน degree out และไม่เหมือน Betweeness ที่เน้นเส้นทางที่สั้นที่สุดระหว่างคู่โหนดในเครือข่าย
